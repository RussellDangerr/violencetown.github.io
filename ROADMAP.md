# Violencetown Roadmap

## Phase 1 — Playable Skeleton ✅
- Tick system (10s deadline, Space to execute)
- Chunk-based shifting city (simplex noise biomes)
- Diagonal movement with squeeze-through
- Canvas tile map renderer
- Basic UI panels

## Phase 2 — NPCs, Combat, Death, The Duke
- NPC spawning and faction AI
- Combat system (melee, ranged, throwables)
- Death = displacement (random respawn, lose some items)
- The Duke as employer/companion
- Faction reputation system

## Phase 3 — Inventory, Crafting, Environment
- 2D grid inventory (holes, fire, acid, theft)
- Tag-based crafting/combinations with UI hints
- Environmental interaction (shove, throw, barricade)

## Phase 4 — Stealth Systems
- Noise/light pollution mechanics
- Sound weapons and distractions (shoestraction)
- Darkness = hiding, deafness/blindness status effects

## Phase 5 — Death Taxi & Missions
- Gas station progression anchor
- Find parts, install at garage, drive
- Duke's taxi missions
- Faction storylines

## Phase 6 — Polish & Entropy
- Entropy events, slow decay
- Day/night cycle
- Audio
- Visual polish

## Future — Godot Migration
- Rebuild in Godot for physics engine, tilemaps, lighting, pathfinding
- Export to HTML5 (WebAssembly) — same browser-based distribution
- Enables complex collision, GPU lighting, shader effects

## Future — Mobile Controls
- Swipe to input direction
- Tap to resolve tick (replaces spacebar)
- Virtual d-pad fallback option
